# 🚀 Инструкция по деплою Marathon Skills

## Шаг 1 — Supabase (база данных)

1. Зайди на **supabase.com** → создай новый проект (бесплатно)
2. Зайди в **SQL Editor** и вставь содержимое файла `supabase_schema.sql` → нажми Run
3. В разделе **Settings → API** скопируй:
   - `Project URL` → это твой `NEXT_PUBLIC_SUPABASE_URL`
   - `service_role` ключ (внизу страницы) → это `SUPABASE_SERVICE_ROLE_KEY`

---

## Шаг 2 — Google OAuth

1. Зайди на **console.cloud.google.com**
2. Создай проект → **APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID**
3. Application type: **Web application**
4. Добавь **Authorized redirect URIs**:
   ```
   https://ТВО-ПРОЕКТ.vercel.app/api/auth/callback/google
   ```
5. Скопируй **Client ID** и **Client Secret**

---

## Шаг 3 — GitHub

1. Создай новый репозиторий на **github.com**
2. Загрузи все файлы этого проекта

---

## Шаг 4 — Vercel

1. Зайди на **vercel.com** → Import Git Repository → выбери свой репозиторий
2. В разделе **Environment Variables** добавь ВСЕ переменные:

| Имя | Значение |
|-----|---------|
| `GOOGLE_CLIENT_ID` | из Google Console |
| `GOOGLE_CLIENT_SECRET` | из Google Console |
| `NEXTAUTH_SECRET` | любые 32+ символа (например: `marathon2026SecretKeyABC123xyz`) |
| `NEXTAUTH_URL` | `https://ваш-проект.vercel.app` |
| `NEXT_PUBLIC_SUPABASE_URL` | из Supabase Settings |
| `SUPABASE_SERVICE_ROLE_KEY` | из Supabase Settings |

3. Нажми **Deploy**

---

## Шаг 5 — Обнови Google OAuth

После деплоя обязательно зайди обратно в Google Console и убедись что redirect URI совпадает с реальным адресом Vercel.
